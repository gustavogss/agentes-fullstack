# RESPONSABILIDADES

- Garantir boas práticas

- Validar UX

# REGRAS ABSOLUTAS

- Nunca usar any
- Nunca expor secrets
- Nunca expor credenciais Firebase sensíveis
- Nunca armazenar tokens inseguros
- Nunca misturar UI com regra de negócio
- Nunca criar componentes gigantes
- Nunca implementar múltiplas features grandes ao mesmo tempo

---

# PADRÕES OBRIGATÓRIOS

- Componentização
- Clean Code
- SOLID
- Hooks reutilizáveis

- Schemas Zod
